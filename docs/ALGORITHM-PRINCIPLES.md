# WizMap 算法原理

> 本文档说明 WizMap 从原始文本到可缩放交互地图的**算法原理**，不涉及具体代码实现。

## 一句话总览

WizMap 的目标是：把**海量高维文本向量**变成一张**可缩放、带地形、带地标**的"地图"。

它用三项核心算法组合实现：

1. **UMAP 降维** —— 高维向量 → 2D 坐标
2. **高斯核密度估计（KDE）** —— 散点 → 连续密度等高线（地形）
3. **四叉树 + 词频主题抽取** —— 空间区域 → 多分辨率语义标签（地名）

三者共用同一套 2D 坐标系，使"点的位置""点的密度""区域的语义"在同一张图上保持一致，并能像地图一样随缩放切换粒度。

---

## 1. 文本向量化（Embedding）

### 原理
用预训练语言模型把每段文本映射成一个高维向量（通常几百到上千维），使**语义相近的文本在向量空间中距离也相近**。向量是整张地图的语义地基——后续所有几何与聚类都建立在"向量距离 ≈ 语义差异"这一假设之上。

### 要点
- 向量本身不直接用于前端渲染，只为降维提供输入。
- 距离度量通常选**余弦相似度**（关注方向而非幅度），契合语义比较的直觉。

---

## 2. UMAP 降维（高维 → 2D）

### 原理
UMAP（Uniform Manifold Approximation and Projection）是一种流形学习降维算法，核心思想是**假设数据采样自一个低维流形**，并尽量在低维空间中复刻高维空间的局部拓扑结构。

算法分两个阶段：

**阶段一：构建高维拓扑图**
- 为每个点找 **k 个最近邻**（k 即 `n_neighbors`）。
- 以这些近邻为边构建一张带权图，权重反映点之间的"亲近度"（距离越近权重越高）。
- 这张图编码了数据的**局部流形结构**。

**阶段二：低维布局优化**
- 在 2D 空间初始化所有点的位置。
- 优化一个目标函数（基于交叉熵），让 2D 布局满足：
  - 高维图里相连的点（近邻）→ 2D 中**相互吸引**
  - 高维图里不相连的点 → 2D 中**相互排斥**（避免重叠）
- 这是一类似弹簧系统的力导向优化。

### 关键参数
| 参数 | 含义 | 影响 |
|---|---|---|
| `n_neighbors` | 构图时的近邻数 k | 大 → 关注全局结构；小 → 关注局部细节 |
| `min_dist` | 2D 中点之间的最小允许距离 | 小 → 点聚得更紧；大 → 点铺得更开 |

### 结果
得到每个点的 (x, y)。语义聚集的文本形成"群岛/大陆"，语义疏远的彼此分离。

但此时的 2D 图只是散点云，缺少**地形感**和**语义注释**——这正是下面两步要补的。

---

## 3. 等高线地形（Gaussian KDE）

### 动机
散点不直观，难以一眼看出"哪里点多"。于是把点云转成**连续密度场**，再画成等高线，让密集区域呈现为视觉上的"山峰"。

### 原理：高斯核密度估计（Kernel Density Estimation）
- 在 2D 平面上铺一个网格（如 200×200）。
- 把每个数据点当作一个**高斯凸起**（钟形曲面）的"源"。
- 对每个网格中心，累加所有高斯凸起在该处的贡献 → 得到该处的**密度估计值**。
- 所有网格点的密度值组成一张光滑的**密度曲面**。

```
点云                  高斯凸起叠加            密度网格 → 等高线
 ·   ·                  ╱╲    ╱╲              ────╲────
  · ·      ──►         ╱  ╲╱╲╱  ╲      ──►     ╲   ╲
 ·   ·                 ╱      ╲╲                ╲___╲
```

### 带宽（bandwidth）的作用
带宽决定高斯凸起的"胖瘦"，是 KDE 最重要的参数：
- **太大** → 过度平滑，所有峰糊成一团，丢失细节。
- **太小** → 噪声多，出现虚假的小峰。

通常根据数据点数自适应设定（样本越多带宽越窄）。

### 加速策略
KDE 在大网格 × 多点上计算开销大，采用**随机子样本拟合**：只从全部点中随机抽取一定数量（如上限 10 万）来估计整体密度分布，用样本近似代替全量。

### 输出
密度曲面的对数值网格矩阵。前端用 marching-squares 等值线算法把它渲染成等高线，视觉上呈现"地形图"，直接回答**"哪里密集"**。

---

## 4. 多分辨率语义标签（Quadtree + 词频主题）

这是 WizMap 最核心的设计——**地图式缩放**：缩得越远，看到大区域的粗粒度主题；放得越大，看到小区域的细粒度主题。类似真实地图"越放大地名越细"的体验。

### (a) 四叉树空间索引

把 2D 平面**递归四等分**：

```
Level 1:  2×2  = 4   块      (大区域，粗粒度)
Level 2:  4×4  = 16  块
Level 3:  8×8  = 64  块
...
Level l:  2^l × 2^l 块        (小区域，细粒度)
```

- 每个叶子块负责一个矩形空间区域。
- 区域内的所有文本聚成一个"文档袋"（bag of documents）。
- **层数 = 分辨率**：浅层对应大区域，深层对应小区域。

四叉树提供了"按空间范围快速聚合文本"的能力——给定任意一块区域，能 O(log n) 拿到它包含的所有点。

### (b) 每块的主题抽取

对每个空间块内的文本做**词频主题分析**：

1. 用 CountVectorizer 统计块内所有文本的词频（同时去除停用词）。
2. 用 TF 类加权选出**该区域最具代表性的若干高频词**。
3. 把 top 词拼成主题名（如 `research-model-learning`）。

这样每个空间块都得到一个"语义标签"，回答**"这个区域讲什么"**。

为处理大词表，词频统计基于稀疏矩阵，只取每块的 top-n 词，避免全量开销。

### (c) 层级自动选择

四叉树理论上可无限细分，但不需要全部计算。算法根据**视图几何**反推需要抽取哪些层：

输入：画布尺寸、最大缩放倍数、期望的瓦片像素宽度（约 35px）。
原理：在某个缩放比例下，算出每一层的瓦片在屏幕上实际占多少像素，选最接近 35px 的那一层。

```
缩放比例 s 下：
  屏幕实际长度 = s × 画布长度
  该层瓦片数 = 2^l
  瓦片像素宽度 = 屏幕实际长度 / 2^l

  → 选使"瓦片像素宽度 ≈ 35px"的 l
```

遍历所有缩放档位，得到 `[min_level, max_level]` 区间，**只对这些层抽取主题**，跳过无意义的全量计算。这保证了：
- 每个标签在屏幕上大约占一块理想大小的区域，既不挤也不空。
- 计算量被严格控制在视图真正会用到的层级上。

### (d) 前端的缩放切换
每个主题标签带有 `(x, y, level)` 三元组。缩放/平移时，前端**只显示当前缩放比例对应层级的标签**，层级间无缝切换——靠的是预先算好的多层级主题，而非实时计算。

---

## 5. 三层合成一张"地图"

| 视觉层 | 数据来源 | 算法 | 回答的问题 |
|---|---|---|---|
| **散点** | 每个点的 (x, y, text) | UMAP | "每条数据在哪" |
| **等高线** | 密度网格 | 高斯 KDE | "哪里密集" |
| **主题标签** | 四叉树各层主题词 | 词频 + 四叉树 | "这个区域讲什么" |

三者**共用同一套 (x, y) 坐标系**，因此：
- 散点落在等高线的"峰"上（密集处）。
- 主题标签标在对应区域的几何中心。
- 任意缩放级别下，三层语义保持一致。

---

## 6. 为什么这套设计"可扩展"

关键在于**预处理与渲染分离**：

- UMAP、KDE、四叉树主题这些**昂贵计算全部在离线阶段一次性算完**。
- 产物只是几个静态文件：密度网格、分层主题标签、散点坐标列表。
- 前端拿到的是**已经分级、已经平滑、已经聚好主题**的结果。
- 缩放/平移交互时，前端只做**坐标变换 + 层级筛选**，不在浏览器里跑任何 ML 计算。

因此即使面对**几十万乃至百万级数据点**，前端交互依然流畅——重活都在上传后的后台管线里做掉了。这正是 WizMap "scalable" 的根本来源。

## 附：算法链路一览

```
原始文本
  │
  │  ① Embedding（预训练模型）
  ▼
高维语义向量  ──────────► "语义相近 ⇒ 距离相近"
  │
  │  ② UMAP（k近邻图 + 力导向优化）
  ▼
2D 坐标 (x, y)  ─────────► "每条数据在地图上的位置"
  │
  ├──────────────────────────────────────┐
  │                                      │
  │  ③ Gaussian KDE（高斯凸起叠加）       │  ④ Quadtree（递归四等分）
  ▼                                      ▼
密度曲面 / 等高线                       多层级空间块
  │                                      │  + 词频主题抽取
  │                                      ▼
  │                                 分层主题标签 (x,y,level,name)
  │                                      │
  └──────────────► 共享 (x,y) ◄──────────┘
                       │
                       ▼
              可缩放交互地图
       （散点 + 地形 + 随缩放切换的地名）
```
