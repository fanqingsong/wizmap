# Upload Dataset 文件格式指南

## 📋 支持的文件格式

Upload Dataset 页面支持以下三种文件格式：

### 1. **TXT 文件** (.txt) - 推荐
纯文本文件，每行一个文本记录。

**示例格式：**
```
机器学习是人工智能的一个分支
深度学习基于神经网络技术
自然语言处理处理文本数据
计算机视觉处理图像数据
数据挖掘发现数据中的模式
```

**使用场景：** 简单的文本集合，如句子、段落、短文本等

### 2. **CSV 文件** (.csv)
逗号分隔值文件，系统会自动读取第一个文本类型的列。

**示例格式：**
```csv
id,text,category
1,机器学习是人工智能的一个分支,技术
2,深度学习基于神经网络技术,技术  
3,自然语言处理处理文本数据,应用
4,计算机视觉处理图像数据,应用
```

**使用场景：** 结构化数据，包含多个字段的数据集

### 3. **JSON 文件** (.json)
JSON 格式文件，支持以下两种结构：

**数组格式：**
```json
[
  "机器学习是人工智能的一个分支",
  "深度学习基于神经网络技术", 
  "自然语言处理处理文本数据",
  "计算机视觉处理图像数据"
]
```

**对象格式：**
```json
{
  "texts": [
    "机器学习是人工智能的一个分支",
    "深度学习基于神经网络技术",
    "自然语言处理处理文本数据",
    "计算机视觉处理图像数据"
  ]
}
```

**使用场景：** 结构化数据，API 返回的数据格式

## 🎯 数据处理流程

上传文件后，系统会自动进行以下处理：

1. **文本解析**: 根据文件格式提取文本内容
2. **嵌入生成**: 将文本转换为向量表示（embeddings）
3. **降维投影**: 使用 UMAP 将高维向量投影到 2D 平面
4. **可视化数据**: 生成 WizMap 可视化所需的数据格式
5. **交互探索**: 在 2D 地图中探索和交互数据点

## 📊 推荐数据集

### 小型数据集 (100-1000 条)
- 适合快速测试和演示
- 处理时间：1-3分钟

### 中型数据集 (1000-10000 条)  
- 适合实际应用
- 处理时间：3-10分钟

### 大型数据集 (10000+ 条)
- 需要较长处理时间
- 可能需要调整资源限制

## ⚠️ 注意事项

1. **文件大小**: 建议单个文件不超过 100MB
2. **文本长度**: 每条文本建议不超过 1000 字符
3. **编码格式**: 使用 UTF-8 编码
4. **数据质量**: 避免空行、重复内容
5. **特殊字符**: 避免特殊控制字符

## 🚀 快速开始

### 1. 准备示例文件
创建一个 `example.txt` 文件：
```
This is a sample text about machine learning
Deep learning uses neural networks for processing
Natural language processing deals with text data
Computer vision focuses on image analysis
Data science combines statistics and programming
```

### 2. 上传文件
- 访问 http://localhost:3001
- 点击 "Upload Dataset" 按钮
- 拖拽或选择你的文件
- 输入数据集名称（可选）
- 点击 "Upload" 开始处理

### 3. 等待处理
系统会自动处理你的数据，完成后会跳转到可视化页面

### 4. 探索数据
在 2D 地图中：
- **缩放**: 鼠标滚轮
- **平移**: 鼠标拖拽
- **选择**: 点击数据点查看文本内容
- **搜索**: 使用搜索框查找特定内容

## 💡 使用建议

- **首次使用**: 从小数据集开始（50-100条）熟悉系统
- **数据准备**: 确保文本内容相关且有意义
- **格式选择**: 根据你的数据源选择最合适的文件格式
- **质量检查**: 上传前检查文件格式和内容质量

上传你的第一个数据集开始探索吧！🎉
